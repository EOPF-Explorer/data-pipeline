# Implementation Plan: T7 Phase 2 — Pyrenees + Alps region rollout

## Overview
Scale the validated single-/few-tile S1 RTC pipeline (CP-A 3/3) to a **region AOI** over the Pyrenees and
Alps: generate a deterministic MGRS tile list for the region (land/DEM-covered), drive it through the
existing pipeline at N=3, and soak it. Start with the **Pyrenees as the starter region** (tractable), then
expand to the **Alps** once the Pyrenees soak is clean. Maps to T7 Phase-2 Task 4 + Task 5 + CP-B.

## Context / current state (verified)
- Pipeline is ready end-to-end (Task 0–3 + #246 deployed; CP-A = 3 tiles concurrent, `aggDEMfail=0`).
- `trigger_cdse.py` already takes `--tiles` as a **comma-separated** MGRS list (`args.tiles.split(",")`),
  `--orbit-direction both` (Task 3, #254). Cron passes `tiles` as a param → no new plumbing for ≤ ~hundreds
  of tiles (the ConfigMap form is Phase-3 Task 7, for ~1000 tiles).
- `eotile` is already a dep (MGRS grid + AOI lookup); `DEM_Union.gpkg` gives land/DEM coverage.
- The `s1rtc` cron is **suspended**; N=3 budget + per-tile mutex are live (Task 1).
- Rough scale: Pyrenees ~10–20 tiles; Alps ~60–90 tiles → combined ~80–110. At N=3, ~1 hr/tile ⇒ Pyrenees
  ~5 hr, Alps ~1–2 days wall-clock. Mountains = full land/DEM coverage (no ocean tiles); the margin fix
  (#262) matters here (swath-edge tiles).

## Architecture decisions
- **AOI = bbox per arc (v1), not a precise mountain polygon.** Pyrenees + Alps bboxes ∩ MGRS ∩ DEM-land.
  A bbox over-includes some non-mountain land (Ebro/Po basins) — but those are valid land tiles, deterministic,
  and need no extra dataset. A precise GMBA/Natural-Earth mountain polygon is a later refinement (OQ-1).
- **Tile list is committed data**, generated by `gen_aoi_tiles.py` and reviewed (a map spot-check) — never
  hand-typed. The generator is pure + deterministic so it's testable and re-runnable.
- **Starter-first rollout:** Pyrenees → validate region flow (CP-B-lite) → Alps. Fail-fast at small scale.
- **Drive via manual batches first, cron second.** For the soak, submit controlled batches (observe cost +
  CDSE throttle) before handing the region to the (unsuspended) cron — more control, same templates.

## Task List

### Task 1 — `gen_aoi_tiles.py` (region → land/DEM MGRS tile list)  *(data-pipeline, TDD)* — M
**Description:** A script that takes a region (named arc → bbox/polygon) and returns the **sorted, deduped**
MGRS tile IDs intersecting it, filtered to tiles with **DEM/land coverage** (a tile is kept iff its swath
cells intersect `DEM_Union.gpkg` land cells — reuse `ensure_dem.py`'s gpkg read + `tiles_for_bbox`/`tile_bbox`).
Regions (Pyrenees, Alps) defined as committed bboxes in the script/config.
**Acceptance criteria:**
- [ ] deterministic, sorted, deduped output for a given region; land-only (every emitted tile has DEM coverage).
- [ ] adversarial: an all-ocean sub-region → **empty list** (no crash); an out-of-Europe polygon → 0 tiles.
- [ ] `--region pyrenees|alps|pyrenees+alps` (or a regions config) → prints/writes the tile list.
**Verification:** `uv run python -m pytest tests/unit/test_gen_aoi_tiles.py` — deterministic fixture polygon →
expected tiles; ocean→empty; every emitted tile resolvable in `DEM_Union.gpkg`.
**Dependencies:** None (reuses ensure_dem helpers + eotile). **Files:** `scripts/gen_aoi_tiles.py`,
`tests/unit/test_gen_aoi_tiles.py` (+ maybe a `regions` constant/config). **Scope:** M

### Task 2 — generate + review the Pyrenees + Alps tile lists  *(data-pipeline)* — S
**Description:** Run the generator for each arc; commit the lists + counts; **spot-check against a map**
(no tiles in the ocean / outside the arc).
**Acceptance criteria:**
- [ ] committed `pyrenees.txt` + `alps.txt` (or a `regions/*.csv`) with counts in the expected ~10–20 / ~60–90 range.
- [ ] map spot-check passes (sample tiles land on the right arc).
**Verification:** visual spot-check (render the tile centroids on a map / cross-ref MGRS); counts sane.
**Dependencies:** Task 1. **Files:** `aoi/pyrenees.txt`, `aoi/alps.txt`. **Scope:** S

### Checkpoint A (after 1–2): selection
- [ ] generator tests green; lists committed + map-reviewed. **Review with human before any processing.**

### Task 3 — Pyrenees starter soak (CP-B-lite)  *(live, staging)* — M
**Description:** Drive the Pyrenees list (~15 tiles) through the pipeline at N=3, **both orbits**, a recent
window — via controlled manual batches against the deployed templates (not the cron yet). Watch render
success, per-tile DEM, throughput, CDSE throttle, dedup (re-run → 0 new).
**Acceptance criteria:**
- [ ] all land tiles render (or cleanly skip empty-data days); per-tile DEM self-provisions; `aggDEMfail=0`.
- [ ] cubes + per-acq items appear for the tiles in `sentinel-1-grd-rtc-tests`; previews 200.
- [ ] a re-run processes **only new** products (no dup slices).
**Verification:** monitor batches to terminal; STAC + titiler spot-check a few tiles; re-run shows 0 new.
**Dependencies:** Task 2, deployed pipeline. **Files:** none (live). **Scope:** M (soak)

### Checkpoint B (after 3): starter region proven
- [ ] Pyrenees soak clean (renders, dedup, cost within budget). **Review before scaling to the Alps.**

### Task 4 — Alps expansion soak  *(live, staging)* — L
**Description:** Same as Task 3 for the Alps list (~60–90 tiles) — the larger soak (~1–2 days at N=3).
Stage in batches; monitor cost/throughput; confirm the margin fix holds across the larger swath variety.
**Acceptance criteria:**
- [ ] Alps tiles render with `aggDEMfail=0` (no new swath-margin gaps — if one appears, it feeds #264/margin);
- [ ] throughput + CDSE cost within budget; dedup correct.
**Verification:** batched monitor; STAC/titiler spot-checks; cost review.
**Dependencies:** Task 3 clean. **Files:** none (live). **Scope:** L (soak — stage in batches)

### Task 5 — point the cron at the region + budget  *(platform-deploy)* — S
**Description:** Once the soaks prove it, set the `s1rtc` cron `tiles` = the committed region list,
`orbit_direction=both`, budget N; keep `suspend: true` until ready, then un-suspend for the region.
**Acceptance criteria:**
- [ ] cron `tiles` references the region list; a scheduled run discovers + processes only **new** products.
**Verification:** `argo lint`; one manual cron-trigger; cube/per-acq items appear for multiple tiles.
**Dependencies:** Tasks 3–4 clean. **Files:** `cronwf/eopf-explorer-cronwf-s1rtc.yaml`. **Scope:** S

### Checkpoint CP-B (complete): region soak
- [ ] 24–48 h soak: only-new processing, no dup slices, cost/throughput within budget → un-suspend for the region.

## Risks & mitigations
| Risk | Impact | Mitigation |
|------|--------|------------|
| AOI bbox over-includes non-mountain land → more tiles than wanted | Med | accept for v1 (valid land tiles); OQ-1 precise polygon later; start with Pyrenees to gauge |
| ~80-tile Alps soak = heavy CDSE download + compute (~1–2 days) | Med | batch it; Pyrenees-first gate; N=3 caps concurrency; watch cost before full run |
| A new swath-margin gap on a mountain tile (like 31TCG) | Med | margin is now 3° (#262); any miss feeds #264 (footprint DEM); soak surfaces it early |
| Comma-separated `tiles` param too long at ~100 tiles | Low | works at this scale; ConfigMap form is Phase-3 Task 7 (~1000 tiles) |
| Un-suspending the cron floods staging | Med | keep `suspend: true` through soaks; un-suspend only after CP-B |

## Decisions (resolved 2026-06-15)
1. **OQ-1 — AOI definition:** ✅ **bbox per arc (v1)**. A precise mountain polygon (GMBA/Natural Earth) is a
   later refinement if the bbox includes too much non-mountain land.
2. **OQ-3 — scope now:** ✅ **Pyrenees first, as a gate.** Build the generator, run only the Pyrenees soak
   (~15 tiles); review at Checkpoint B before committing to the ~80-tile Alps soak.
3. **OQ-4 — window:** ✅ **recent N-day window** (matches the real-time cron use case; exercises dedup vs
   fresh products). Exact N picked at soak time (default last 7–14 days).
